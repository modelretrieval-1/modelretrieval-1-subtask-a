## Overview

Sub Task A focuses on Pre-trained BERT Model Retrieval for text classification tasks. This repository provides:

- task and candidate model's metadata
- data of training tasks
- CLI for downloading test task's data and candidate model's artifacts.
- ground-truth relevance scores of task-model pair for training tasks
- a baseline notebook and utility functions

## Quick Start

1. Install uv:
   https://docs.astral.sh/uv/getting-started/installation/
2. Sync dependencies:

   ```bash
   uv sync
   ```



3. Download models from `data/models.csv`:

   Run command help:

   ```bash
   uv run python main.py download_models --help
   ```

   Download selected model IDs (repeat `--model-id`):

   ```bash
   uv run python main.py download_models --model-id 1 --model-id 2 --model-id 20
   ```

   Download all models listed in `data/models.csv`:

   ```bash
   uv run python main.py download_models --all
   ```

   Rules:

   - Use either `--all` or `--model-id` (not both).
   - One of them is required.

   Downloaded files are stored under `data/models/{model_id}` with zero-padded folder names (for example: `01`, `02`, ..., `20`).

4. Download test tasks from `data/tasks.csv`:

   Download all test tasks listed in `data/tasks.csv`:

   ```bash
   uv run python main.py download_test_tasks --all
   ```

   Downloaded files are saved under `data/task-data/test-tasks/{task_id}` with zero-padded folder names such as `0001`, `0002`, and `0048`.
   Each task folder contains `train.jsonl` and `val.jsonl`.

   - Optional

      Download selected test task IDs (repeat `--task-id`):

      ```bash
      uv run python main.py download_test_tasks --task-id 1 --task-id 2 --task-id 48
      ```
      
      Run command help:

      ```bash
      uv run python main.py download_test_tasks --help
      ```



      Notes: Use either `--all` or `--task-id` (not both).



5. (Optional) Open the baseline notebook:

   ```text
   notebooks/baseline-avg-ranking.ipynb

### Key Components

- data/tasks.csv: Task's metadata
- data/models.csv: Candidate model's metadata
- data/ground-truth.csv: Relevance scores for each task-model pair (train tasks only)
- data/models: Downloaded model artifacts (**Note: Have to download using CLI**)
- data/task-data/train-tasks: Data for training tasks
- data/task-data/test-tasks: Data for test tasks (**Note: Have to download using CLI**)
- notebooks/baseline-avg-ranking.ipynb: Baseline ranking workflow
- utils/__init__.py: DCG and nDCG utility functions

## Data Dictionary

### tasks.csv

| Column | Description |
|---|---|
| task_id | Task identifier |
| task_name | Task name |
| task_source | Source type: hf (Hugging Face) or llm (LLM-generated) |
| task_class_count | Number of class labels |
| task_type | train or test |
| task_hf_dataset_name | Hugging Face dataset name (if applicable) |
| task_hf_subset_name | Hugging Face subset name (if applicable) |
| task_hf_url | Hugging Face dataset URL |
| task_instruction | Brief description of the task |
| task_criticalness | High, Medium, or Low task sensitivity |

### models.csv

| Column | Description |
|---|---|
| model_id | Model identifier |
| model_name | Model name |
| model_hf_url | Hugging Face model URL |

### ground-truth.csv

| Column | Description |
|---|---|
| task_id | Task identifier |
| model_id | Model identifier |
| f1 | F1 score after fine-tuning on task training data and evaluating on task test data |
| model_rank | Rank by F1 within the task (lower is better) |
| normalized_f1 | F1 normalized by the best F1 on the same task |
| relevance_score | Relevance score derived from normalized F1 |


## Submission Format

Use **TREC_EVAL** format:

```text
topicID Q0 docID Rank Score RunID
```

- topicID: query task ID
- Q0: fixed string (Q0)
- docID: model ID
- Rank: ranking position of the model
- Score: predicted score for the model
- RunID: your run identifier

### Example

```text
1 Q0 1 1 0.99 Run01
2 Q0 7 2 0.95 Run01
...
```

## Evaluation Metrics

- nDCG@k where k = 1, 3, 5
